## 二、opencode（AI Coding Agent 引擎）

### 基本信息
- **仓库地址**：https://github.com/anomalyco/opencode
- **版本**：v1.17.8（npm 包名 `opencode-ai`）
- **包管理器**：Bun（monorepo，使用 turborepo）
- **许可证**：MIT
- **描述**："The open source AI coding agent"

### 安装方式
```bash
curl -fsSL https://opencode.ai/install | bash
npm i -g opencode-ai@latest
scoop install opencode        # Windows
choco install opencode        # Windows
brew install opencode         # macOS/Linux
```

### 架构
- **Monorepo**，主要 packages 包括：
  - `packages/core` — 核心引擎（配置、插件、权限、模型、文件系统、PTY 等）
  - `packages/cli` — CLI 入口
  - `packages/opencode` — 主程序入口
  - `packages/tui` — 终端 UI
  - `packages/app` — Web 应用
  - `packages/desktop` — 桌面应用（Tauri）
  - `packages/console` — 控制台后端
  - `packages/plugin` — 插件 SDK
  - `packages/sdk` — Python/JS SDK
  - `packages/llm` — LLM 接口层
  - `packages/server` — 服务器端
  - `packages/ui` — UI 组件库
  - 还有 `identity`, `effect-drizzle-sqlite`, `function`, `stats`, `slack`, `storybook` 等

### 内置 Agent
- **build** — 默认的全权限 Agent，用于开发工作
- **plan** — 只读 Agent，用于分析和代码探索
- **general** — 子 Agent，用于复杂搜索和多步骤任务

### 核心能力
- 终端 TUI 交互
- 插件系统（`.opencode/plugins/`）
- MCP（Model Context Protocol）支持
- 技能系统（`.opencode/skills/`）
- 主题系统（`.opencode/themes/`）
- 权限控制
- 多模型支持
- 内置免费 Qwen Plus（alibaba/qwen-plus）
- 非交互模式：`opencode run --dangerously-skip-permissions`

### 配置文件
- `.opencode/opencode.jsonc` — 项目级配置

### 核心亮点功能

#### 🔄 Ralph Loop / ulw-loop
- **核心思想**：每次 session 空闲时自动检查 assistant 输出有没有 `<promise>DONE</promise>`，没有就注入 continuation prompt 让 AI 继续干活
- "没干完就继续" — 不需要人工介入

#### 🪄 ultrawork
- 一个命令激活所有 Agent，不完成不停止

#### 👥 Team Mode（v4.0）
- 1 个主导 Agent + 最多 8 个并行成员
- tmux 可视化
- 内置 skills: `hyperplan`（5 个敌意批评者审查计划）、`security-research`（漏洞挖掘）

#### 🔗 Hash-Anchored Edit Tool（Hashline）
- 每行代码附带内容哈希标签（`11#VK| function hello()`）
- 编辑时通过哈希引用，文件变更后哈希不匹配则拒绝编辑
- 从 6.7% → 68.3% 成功率提升（Grok Code Fast 1 benchmark）

#### 🛠️ LSP 集成
- `lsp_rename`, `lsp_goto_definition`, `lsp_find_references`, `lsp_diagnostics`

#### 🔎 AST-Grep
- 25+ 语言的模式感知代码搜索和重写

#### 📐 规则注入
- AGENTS.md 和 `.omo/rules/**` 自动注入到 Agent 上下文

#### 🧠 后台 Agent
- 可以同时启动 5+ 个专业化 Agent 并行工作

#### 🧬 Skill-Embedded MCPs
- Skills 自带 MCP 服务器，按需启动，用完即走

## 四、opencode-novel-loop（小说对话标注工具）

### 基本信息
- **路径**：`E:\projects\opencode-novel-loop`
- **版本**：v0.1.0
- **项目名**：`dialoop`（Python 包名）
- **许可证**：MIT
- **Python 版本要求**：>= 3.8

### 定位
> 结合 OpenCode + Ralph Loop 的零配置小说对话说话人标注工具

### 背景故事
作者有一批小说文本需要标注——把文中所有 `「对话内容」` 标注上对应的说话角色名。原始方案是：
1. 服务器上装 OpenCode
2. 装 openloop 插件（ralph-loop）
3. 一条提示词触发自动循环

ralph-loop 的巧妙设计：每次 session 空闲时检查 assistant 输出有没有 `<promise>DONE</promise>`，没有就注入 continuation prompt。

但原始方案门槛太高，所以要做成独立 CLI 工具。

### 当前状态
🚧 项目已启动但处于早期阶段。

## 五、三个项目的关联

```
                    ┌──────────────────────────┐
                    │      opencode (v1.17.8)   │
                    │   AI Coding Agent 引擎    │
                    │   （底层基础设施）         │
                    └──────────┬───────────────┘
                               │ 插件系统
                    ┌──────────▼───────────────┐
                    │ oh-my-openagent (v4.11.1)│
                    │   OmO 全家桶插件          │
                    │   （增强层）              │
                    └──────────┬───────────────┘
                               │ Ralph Loop 灵感
                    ┌──────────▼───────────────┐
                    │ opencode-novel-loop      │
                    │   dialoop 小说标注工具    │
                    │   （应用层）             │
                    └──────────────────────────┘
```

1. **opencode**（核心引擎）→ 底层 AI 编程 Agent 框架，提供模型推理、TUI、插件系统
2. **oh-my-openagent**（增强插件）→ 基于 opencode 插件系统的全家桶，提供 ultrawork、Team Mode、LSP、Hashline 等高级功能
3. **opencode-novel-loop**（应用）→ 最初基于 opencode + openloop 插件做小说标注，后来演进为独立 Python CLI 工具 `dialoop`

### 关键灵感传承
- **Ralph Loop**（openloop 插件）→ 自动循环机制：检测是否完成，未完成则继续
- **oh-my-openagent** 将其发展为 `ulw-loop` 和 `ultrawork`
- **opencode-novel-loop** 的 `agent_loop.py` 用 Python 重写了类似的循环逻辑
